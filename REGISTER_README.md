# 批量注册功能使用说明

## 安装浏览器驱动（首次使用前）

在启动项目前，需要先安装浏览器驱动：

```bash
# 在项目根目录运行（安装两种浏览器）
python backend\install_browsers.py

# 或者只安装推荐的 patchright
python backend\install_browsers.py --browser patchright
```

**注意**：浏览器驱动大小约几百 MB，请确保网络连接稳定。

## 启动项目

安装浏览器驱动后，可以通过以下命令启动完整项目：

```bash
npm run dev
```

这会同时启动：
1. 后端服务（端口 8765）
2. UI 开发服务器
3. Desktop 应用

或者单独启动后端：

```bash
# 在项目根目录
npm run dev:backend

# 或者直接运行
cd backend
python run.py
```

## 新功能使用

### 1. 配置 OAuth2（可选）

1. 点击侧边栏的"设置"
2. 在 OAuth2 设置中填写：
   - Client ID：Azure 应用程序的 client_id
   - Redirect URL：如 http://localhost:8000
   - Scopes：默认已填写常用权限

### 2. 配置代理池（推荐）

1. 点击侧边栏的"设置"
2. 在代理池管理中添加代理地址
3. 格式如：`http://127.0.0.1:7897`
4. 可以添加多个代理，注册时会随机选择

### 3. 开始批量注册

1. 进入"账号池"页面
2. 点击"批量注册"按钮
3. 配置注册参数：
   - 浏览器引擎：推荐 Patchright
   - 并发数：建议 5
   - 注册数量：单次注册的数量
   - 机器人检测等待时间：建议 11 秒
   - 验证码最大重试次数：建议 2 次
   - 获取 OAuth2 Token：勾选后会使用全局配置获取 token
4. 点击"开始注册"
5. 查看进度面板，实时显示注册状态
6. 注册成功后会自动添加到账号池

## 注意事项

1. **代理重要性**：同一 IP 短时间内多次注册会被限制，建议使用代理池
2. **浏览器驱动**：首次启动会自动安装浏览器驱动（约几百 MB）
3. **验证码**：当前只支持按压式验证码
4. **并发控制**：并发数不宜过大，建议 5-10 个
5. **注册数量**：单 IP 建议一次不超过 5 个

## 故障排查

### 后端启动失败

1. 确保已安装 Python 3.11+
2. 确保已安装依赖：`pip install -r backend/requirements.txt`
3. 检查端口 8765 是否被占用

### 注册失败

1. 检查代理是否可用
2. 检查浏览器驱动是否安装成功
3. 查看控制台错误信息
4. 尝试降低并发数

### 验证码无法通过

1. 增加机器人检测等待时间
2. 增加验证码重试次数
3. 检查 IP 质量
