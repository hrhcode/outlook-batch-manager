# 批量注册页面改造方案

## 一、需求变更说明

### 1.1 原方案问题
原方案将批量注册功能作为账号池页面的弹窗和面板组件，与账号池功能耦合度较高。

### 1.2 新方案目标
1. **批量注册作为独立页面**：拥有完整的页面路由和独立布局
2. **配置集中管理**：将OAuth2配置、代理池配置迁移到批量注册页面
3. **清晰的导航层级**：批量注册页面在账号池页面上方

## 四、详细设计

### 4.1 批量注册页面 (register-page.tsx)

```typescript

  const [activeTab, setActiveTab] = useState("task");
  
  return (
    <div className="register-page">
      <PageHeader title="批量注册" action={<StartButton />} />
      
      <Tabs value={activeTab} onValueChange={setActiveTab}>
        <TabsList>
          <TabsTrigger value="task">注册任务</TabsTrigger>
          <TabsTrigger value="oauth2">OAuth2配置</TabsTrigger>
          <TabsTrigger value="proxy">代理池管理</TabsTrigger>
          <TabsTrigger value="advanced">高级设置</TabsTrigger>
        </TabsList>
        
        <TabsContent value="task">

        <TabsContent value="proxy">
          <ProxyConfigPanel />
        </TabsContent>
```

### 4.3 注册任务面板 (register-task-panel.tsx)

包含以下功能区域：

1. **配置区域**
   - 注册数量输入（number input）
   - 并发数滑块（slider）
   - 浏览器选择（radio/select）
   - 是否获取Token（checkbox）

2. **操作区域**
   - 开始注册按钮（主按钮）
   - 停止注册按钮（危险按钮，注册中时显示）

3. **进度区域**
   - 进度条
   - 统计卡片（总数/成功/失败）
   - 实时日志/状态

4. **结果区域**
   - 成功账号列表（可导出）
   - 失败记录（可查看原因）

## 七、注意事项

1. **数据持久化**：OAuth2配置和代理池配置仍通过原有API保存到后端
2. **状态管理**：注册任务的状态在页面内管理，刷新页面会丢失当前进度
3. **用户体验**：提供清晰的标签切换，避免配置过于集中导致混乱
4. **响应式设计**：确保在较小屏幕上也能正常使用标签页
