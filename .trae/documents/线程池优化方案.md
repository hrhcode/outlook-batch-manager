# 线程池优化方案

## 一、当前问题分析

### 问题1：僵尸线程池（未使用）
```python
# register_service.py 第30行
self.executor = ThreadPoolExecutor(max_workers=10)  # 创建但从未使用
```

**影响：** 占用系统资源，代码可读性差

### 问题2：每批任务重复创建线程池
```python
# _run_task() 方法中
while current < config.max_tasks:
    with ThreadPoolExecutor(max_workers=config.concurrent_flows) as executor:
        # 每批都创建新线程池
        pass  # 批次结束销毁线程池
    await asyncio.sleep(0.5)
```

**影响：**
- 线程创建/销毁开销（约 50-100ms/次）
- 系统资源波动
- 任务间隔期间线程资源浪费

---

## 二、优化目标

1. 删除未使用的僵尸线程池
2. 复用线程池，减少创建/销毁开销
3. 保持并发控制粒度（每批任务数可控）
4. 确保任务取消功能正常工作

---

## 三、优化方案

### 方案A：任务级线程池复用（推荐）

**思路：** 每个注册任务创建一个线程池，任务内所有批次复用同一个线程池

```python
class RegisterService:
    def __init__(self, event_bus: EventBus):
        self.event_bus = event_bus
        self.active_tasks: dict[int, bool] = {}
        # 删除 self.executor

    async def _run_task(self, task_id: int, config: RegisterConfig):
        # 任务开始时创建线程池
        with ThreadPoolExecutor(max_workers=config.concurrent_flows) as executor:
            while current < config.max_tasks:
                # 复用同一个线程池
                futures = []
                for _ in range(batch_size):
                    future = loop.run_in_executor(executor, run_register)
                    futures.append(future)
                
                # 等待本批次完成
                for future in futures:
                    result = await future
                
                await asyncio.sleep(0.5)
        # 任务结束时销毁线程池
```

**优点：**
- 线程池只创建1次，销毁1次
- 代码改动小，逻辑清晰
- 支持任务取消（通过 active_tasks 标记）

**缺点：**
- 任务执行期间线程池一直占用内存

---

### 方案B：全局线程池（备选）

**思路：** 整个 RegisterService 共享一个线程池

```python
class RegisterService:
    def __init__(self, event_bus: EventBus):
        self.event_bus = event_bus
        self.active_tasks: dict[int, bool] = {}
        self.executor = ThreadPoolExecutor(max_workers=20)  # 全局线程池
    
    def shutdown(self):
        self.executor.shutdown(wait=True)
```

**优点：**
- 线程池只创建1次
- 多任务共享线程资源

**缺点：**
- 并发控制复杂（需要信号量控制同时执行数）
- 任务间相互影响
- 不适合当前"每任务独立控制并发"的需求
